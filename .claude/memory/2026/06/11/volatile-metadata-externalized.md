# Volatile Metadata Externalized

Memory file frontmatter now contains content fields only: `name`, `description`, `metadata.type`.
Volatile metadata (`accessed`, `count`, `tier`, `dropped`) lives in gitignored
`_meta.json` per date directory — same directory as the `.md` files.

**Why:** Every touch/prune/promote dirtied 15+ git-tracked files with pure metadata churn.
Now git only tracks memory content; device-local state self-heals from path dates.

## Key design decisions

- `_meta.json` per `YYYY/MM/DD/` directory, keyed by filename (not full path)
- `created` date derived from path — no longer in frontmatter
- `MEMORY.md` is fully generated by `rebuildIndex()`, gitignored, device-local
- `scope-validate.mjs` runs at SessionStart (`--fix`) to ensure intermediate file integrity
- `scanAllScopes()` provides multi-scope task display via `/todo report`
- Scope discovery is purely filesystem-driven — any directory with `.claude/memory/` is a scope
- `findAllScopes().forEach(rebuildIndex)` for multi-scope rebuilds; `rebuildIndex` is single-scope

## What changed

- `rem/lib.mjs`: removed frontmatter-based `getTier`/`setTier`/`hasAllFields`/`stampMissingFields`/`bumpAccessed`/`getAccessCount`/`updateIndexAccessed`; added `loadMemoryState`/`saveMemoryMeta`/`getMemoryMeta`/`bumpAccessed`(state)/`dropFromIndex`/`rebuildIndex`/`findChildScopes`
- All scripts updated: stamp (simplified), prune (state-based), touch (_meta.json), rem-prep (state), compact (state), task-engine (multi-scope report), task-lib (scanAllScopes/formatScopeReport)

- `shared/lib.mjs`: `reviewFrontmatter` no longer emits `created`/`accessed`/`tier`
- `.gitignore`: added `MEMORY.md` + `**/_meta.json` to both repos
- 233 tests pass (all rem + sharp-review), 2 new test files (memory-state, scope-validate)
- 3 new rules: `hook.md`, `providers.md`, `migration.md`; 3 updated: `cc-market-repo.md`, `git-conventions.md`, `skills-location.md`
